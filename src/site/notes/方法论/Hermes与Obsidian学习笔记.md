# Hermes Agent 与 Obsidian 学习笔记

> 运营阶段顺手搭了一套工具链：Obsidian 做知识管理 + 数字花园，Hermes Agent 做 AI 助手 + 自动化。

---

## Obsidian

### 为什么选 Obsidian

之前工作记录散落在各个地方——飞书文档、本地 Word、微信聊天记录。找个东西要翻半天。Obsidian 的核心吸引力就三点：

1. **本地 Markdown 文件** — 数据在自己硬盘上，不怕平台倒闭
2. **双向链接** — `[[这样]]` 就能把笔记连成网，不用分类目录
3. **Digital Garden 插件** — 一键把笔记部署成公开网站

### 怎么用的

- **知识库**：用 LLM Wiki 的三层结构（项目档案 / 方法论 / 素材库），所有工作相关的东西往里扔
- **数字花园**：通过 Digital Garden 插件部署到 [oscar-notebook.netlify.app](https://oscar-notebook.netlify.app)，公开可见，述职时当 PPT 讲
- **周报 Skill**：把周报分析流程写成 Skill（见 [[skills/照明周报全品类分析\|照明周报 Skill]]）

### 踩过的坑

- **Git 推送**：命令行 token 经常过期，Digital Garden 插件自带的推送更稳定
- **中文文件名**：Obsidian 对中文支持很好，但 wikilink 重命名后要全局替换
- **Netlify 构建**：push 后 1-2 分钟部署完成，偶尔需要手动触发

## Hermes Agent

### 是什么

一个运行在本地的 AI Agent，能执行终端命令、读写文件、搜索网页、控制浏览器。和 Claude Code 的区别是它能持久运行（cron job），不只是交互式。

### 做了什么

在运营阶段用 Hermes 干了这些事：

1. **周报自动化**（见 [[方法论/周报工作流\|周报工作流]]）
   - 从原始 CSV 数据 → 品类结构分析 → 健康度信号灯 → 异常诊断
   - 把每周重复的劳动交给 Agent，我只需要看最终结果 + 写分析

2. **文案审计**（见 [[skills/sealight-copy-audit\|文案合规审计]]）
   - 自动检查英文营销文案的合规问题
   - 四层审计：品牌调性、语法拼写、合规红线、转化优化

3. **图片提示词生成**（见 [[skills/sealight-image-prompts\|素材提示词]]）
   - 根据产品规格自动生成 AI 图片提示词
   - 统一视觉风格和品牌元素

4. **代码审查可视化**（见 [[skills/HTML代码审查可视化\|HTML 代码审查]]）
   - 生成自包含的 HTML 审查报告，方便和非技术同事沟通

5. **本次述职笔记整理**
   - 跨三个文件夹（转正制作 / 运营 / Cowork专用）扫描资料
   - 批量重命名目录 + 全局更新 wikilinks
   - 生成 8 篇结构化的述职笔记

## 工具链全景

```
数据层               分析层              输出层
───────           ───────            ───────
PDM/BI 导出  →   Hermes Skill   →   周报表格
购物广告 CSV →   自动分析       →   健康度诊断
退货原因 CSV →   交叉诊断       →   异常定位
                              →   Obsidian 笔记
                              →   Netlify 数字花园
```

---

## 学到了什么

- **先跑通手工流程，再自动化**：不要一上来就写脚本。先把事情做对，再让 AI 帮你做快
- **Skill 是资产的积累**：每次写一个 Skill 就相当于给未来的自己省时间。三五个 Skill 积累下来，效率提升是指数级的
- **公开写作倒逼质量**：笔记发到公网上，自然会认真写。半成品也比不写强
- **AI 是杠杆，人是支点**：Hermes 能跑数据、写文案、审合规，但判断「这个数字要不要深挖」「这句话用户看了会不会买」只有人能做
