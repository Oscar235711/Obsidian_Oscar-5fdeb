# 周报工作流

> 从手工拼数据到半自动化，14 周周报的进化过程。

---

## 做了什么

接手照明事业部周报后，搭建了一套从原始数据到最终报告的半自动工作流。

### 周报覆盖范围

- **周期**：W12（3月22日）— W27（7月4日），共 14 周
- **数据源**：P&L 损益数据、购物广告数据、退货原因明细
- **输出**：全品类总览 + 健康度信号灯 + 异常诊断 + 周报总结

### 数据流程

```
原始数据导出（PDM/BI）
  → 品类结构分析表
  → MTD 汇总
  → 购物广告分析
  → 退货原因明细
  → 健康度信号灯
  → 异常诊断
  → 最终周报
```

### 自动化程度

早期全手工——从 BI 导出数据、Excel 透视、手动填表、手动写分析。

后来逐步把重复环节自动化：
- 品类结构分析 → CSV 脚本处理
- MTD 汇总 → 半自动公式
- 购物广告分析 → R4 补充数据源
- 退货原因明细 → R5 补充数据源

最终建成了 [[skills/照明周报全品类分析\|照明周报 Skill]]，让 Hermes Agent 能自动跑完大部分流程。

---

## 学到了什么

- **自动化不是一次做完的**：先跑通手工流程，再找最痛苦的环节优先自动化
- **数据源比算法重要**：购物广告和退货原因是后来才加入的深度数据源，加入后诊断准确度大幅提升
- **诊断聚焦大体量**：占品类 >5% 的大系列重点看，尾部小系列波动不用纠结
- **周报不是交作业**：好的周报能让看的人 5 分钟抓住重点，而不是一堆数字罗列
