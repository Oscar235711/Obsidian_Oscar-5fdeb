## 二、阶段 A：xlsx 生成规范

### 2.1 结构

Sheet `全品类总览`，31 行 × 10 列（A=品类, B=指标项, C=销售额, D=毛利, E=毛利率, F=R1, G=R2, H=R3, I=R4, J=R5）

每品类 6 行：

| 指标项 | 数据来源 |
|--------|---------|
| 本周 | MTD W{NN} 聚合 + R5 取自周报 CSV |
| 上周 | MTD W{NN-1} 聚合 + R5 取自周报 CSV |
| 周环比 | 公式（金额列增长率，比率列差值） |
| 月数据 | MTD W23+W24+W25+W26 聚合 |
| 月目标 | 季度不变（从历史 xlsx 继承） |
| 月完成率 | 公式 = 月数据/月目标 |

### 2.2 R1-R5 公式

```python
R1 = 材料成本-实际 / 销售额
R2 = 物流成本 / 销售额           # MTD 已有物流成本列
R3 = (促销折扣+仓储费+店铺费用-其它+佣金) / 销售额
R4 = 退款金额 / 销售额
R5 = (广告SP+店铺余额+站外推广) / 销售额  # MTD 近似
```

> MTD 无 `推广合计`，`推广总计` 含促销折扣不可用。「本周」「上周」行的 R5 用周报 CSV 精确值。

### 2.3 月目标值（6月，季度不变）

| 品类 | 销售额 | 毛利润 | 毛利率 | R1 | R2 | R3 |
|------|--------|--------|--------|-----|-----|-----|
| 大灯 | 3,119,667 | 633,179 | 20.30% | 19.05% | 10.17% | 7.70% |
| HID | 291,374 | 54,337 | 18.65% | 15.95% | 11.75% | 6.21% |
| 雾灯 | 721,086 | 156,779 | 21.74% | 17.36% | 10.95% | 8.46% |
| 小灯 | 661,942 | 153,546 | 23.20% | 17.29% | 13.15% | 6.67% |
| CA | 139,698 | 20,407 | 14.61% | — | — | — |

### 2.4 条件格式

**周环比行**（第 4/10/16/22/28 行）：

| 列 | 方向 | 阈值 |
|----|------|------|
| C/D/E | ↑绿 ↓红 | \|Δ\|>3% |
| F/G/H/J | ↓绿 ↑红 | \|Δ\|>3% |
| I (R4) | ↓绿 ↑红 | \|Δ\|>2% |

**月完成率行**（第 7/13/19/25/31 行）：

| 列 | 达标 | 不达标 | 依据 |
|----|------|--------|------|
| C/D | > 时间进度 | < 时间进度 | W26=90%，W25=67%，W24=43%，W23=20% |
| E | > 95% 目标值 | < 95% | 固定 |

### 2.5 防坑：R5 店铺段落混淆

周报 CSV 内含 `Sealight` → `Sealight-CA` 两段，同名品类（大灯灯泡）出现两次。**必须用 `current_store` 变量追踪段落**，否则 CA 大灯 R5（~7%）会错误覆盖主站（~36%）。

## 三、阶段 C：HTML 报告规范

### 3.1 分析优先级

- **R1-R3**：各品类一句话带过
- **R4**：深度分析，交叉引用 `退货原因明细报表` → 品类×TOP1退货原因表
- **R5**：深度分析，交叉引用 `购物广告分析` → 花费/ROI/CTR/CVR 周环比表

### 3.2 HTML 五节结构

| 节 | 内容 |
|----|------|
| 1 本周主线 | 大盘一句话（销售额/毛利率/R4/R5 + 核心驱动 + 主要矛盾） |
| 2 R4 退货分析 | 品类×R4 表 + 核心发现（引用退货原因报告） |
| 3 R5 推广分析 | 品类×R5 表 + 购物广告子表（引用购物广告报告） |
| 4 信号灯 | 5 行表（品类/状态/定性/关注板块） |
| 5 异常诊断 | a 大灯+HID 6-7 条 / b 雾灯+小灯 5-6 条 / c CA 2-3 条 |

### 3.3 设计系统

```css
--ink:#1f2329; --sub:#4e5969; --line:#e5e6eb;
--bad:#c0392b; --good:#2e7d32; --warn:#b88600;
--bad-bg:#fdecea; --good-bg:#eaf5ec; --warn-bg:#fbf3e0;
body: 14px, #f5f6f7; .wrap: max-width 920px, 白色卡片
h1: 22px; h2: 17px; h3: 14.5px
table: 13.5px; .badge: 12.5px pill
.good/.bad: 行内颜色; .grp: 加粗组名
```

### 3.4 诊断规则

- 只分析体量 > 品类 5% 的系列，尾部小系列不入诊断
- 同类问题打包成「组」
- R4/R5 异常必须交叉引用对应报表
- **不写「建议：」**，只描述现象+数据

## 五、风格铁律

2. ✅ 数字用 `↑↓` 箭头

## 六、常见坑

1. CSV 编码：内部文件 `gb18030`，周报输出 CSV 是 `UTF-8 BOM`
2. `推广总计` 含促销折扣不可用，用 `推广合计`
3. R5 店铺段落混淆：周报 CSV 分 Sealight / Sealight-CA 两段
4. 月完成率公式防除零
5. 退货明细不完整，表述用「占明细 X%」而非「占总退款 X%」
6. CA 月目标无 R1-R3，列留空
7. 非 SEALIGHT 品牌购物广告仅作横向参考
8. 时间进度逐周变化，条件格式阈值需逐周检查
