## 第一课：进阶赋能型目录广告（Advantage+ Catalog Ads）

### 原理

Meta 获取你的商品目录 → 根据用户行为自动匹配最可能购买的商品 → 展示给每个用户不同的商品。

**传统广告：** 你手动选商品 × 手动选受众
**目录广告：** Meta 自动匹配商品 × Meta 自动找受众

### 四种目录广告类型

| 类型 | 触发条件 | Sealight 适用 |
|------|---------|-------------|
| DPA（动态产品广告） | 用户在你网站看过某商品 | ✅ 访客召回 |
| DABA（广泛受众动态广告） | 没来过你网站的新用户 | ✅ 拉新 |
| Catalogue & Store | 线上+线下门店 | ❌ |
| **Advantage+ Catalog** | AI 自动优化进阶版 | ✅ 当前已开启 |

### 当前问题

## 第二课：数据集（Meta Pixel vs CAPI）

### 现状

Sealight 仅有 **Meta Pixel** 一个数据集，缺少 Conversions API（服务器端）。

### 对比

| 数据层 | 只有 Pixel | Pixel + CAPI |
|--------|-----------|-------------|
| 浏览器事件（浏览/加购） | ✅ | ✅ |
| 服务器端事件（购买确认）| ❌ | ✅ |
| 广告拦截器/隐私模式用户 | ❌ 丢失 | ✅ 绕过 |
| iOS Safari 用户 | 大部分丢失 | ✅ 补回 |

### 影响

1. **ROI 虚低** — 购买完成但 Pixel 没抓到，Meta 无法归因
2. **目录数据质量差** — 服务端购买事件缺失 → 商品级转化数据不全
3. **相似受众质量打折扣** — 种子数据本身缺失

## 第四课：竞拍策略与预算优化

### 竞拍机制

Meta 广告位竞价：

```
你的出价 × 预估行动率 + 广告质量 = 总价值
```

最高总价值赢，不是最高出价赢。

### 三种出价策略

| 策略 | 适合 | 建议 |
|------|------|------|
| **最低成本** | 新手/学习期 | ✅ 前三周用这个 |
| 成本上限 | ROI 导向 | 学习期结束后切换 |
| 出价上限 | 精细控制 | 暂不需要 |

### 预算模式

| 方式 | Sealight 现状 |
|------|-------------|
| CBO（广告系列预算优化） | ✅ 已选用 |
| 广告组预算 | 暂不需要 |

## 第五课：拉新配置

### 不需要改目标

拉新电商广告最佳目标仍然是「销量」—— Meta 找最可能购买的**新用户**。

真正的拉新配置在**受众层**。

### 三种拉新方案

| 方案 | 配置 | 适合 |
|------|------|------|
| **Advantage+ 购物广告** | Meta 全自动，你只需给预算+素材 | ✅ 推荐首选 |
| 手动受众 | 自定义兴趣/行为/排除 | 可控但周期长 |
| 相似受众（Lookalike） | 基于客户数据生成相似人群 | 有足够种子数据后 |

## 第八课：学习期

### 定义

上线后前 7 天（或 50 个优化事件）为学习期。Meta 在试错，数据波动正常。

### ❌ 学习期内禁止

- 改预算超过 20%
- 改受众
- 停掉再开
- 频繁换素材
- 关掉「表现差」的广告组

### ✅ 学习期内可以

- 看消耗和 CPA 趋势（单日波动忽略）
- 小幅调整出价上限
- 加素材（不重置学习期）
- 跑了两天花费 $0 → 出价太低或受众太窄，需调整

### 学习期完成标志

广告组状态从「学习中」→「投放中」

## 关键概念速查

| 概念 | 含义 |
|------|------|
| **Pixel** | 网站端追踪代码，浏览器事件 |
| **CAPI** | 服务器端 API，绕过广告拦截 |
| **CBO** | 广告系列预算优化，Meta 自动分配 |
| **DPA** | 动态产品广告，做过什么推什么 |
| **Lookalike** | 相似受众，找你客户的相似人群 |
| **学习期** | 上线初期的算法优化阶段 |
| **归因窗口** | 点击/浏览后多久内算转化 |
| **高级匹配** | 抓取用户填表信息增强匹配 |
