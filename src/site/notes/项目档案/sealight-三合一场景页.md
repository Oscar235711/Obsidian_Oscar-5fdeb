# Sealight 三合一场景页重构

> 2026-05-12 将原型推倒重来，三个独立场景页合并为一个综合性场景解决方案页面。

## 重构决策

### 触发原因
- Refresh 场景经讨论后人群太窄
- 需要重新选定场景
- 原 prototype 的独立场景页架构需要调整

### 定调
- **核心定位**：为用户提供解决场景问题的方案（而非产品列表）
- **三合一**：三个场景合并到一个页面
- **布局**：纵列（用户否决横列，认为纵列更符合场景页逻辑）

## 设计决策

### 布局：纵列 vs 横列
- **用户选择**：纵列
- **理由**：纵列更符合场景页"用户沿认知路径向下滚动"的逻辑

### 内容转化原则
- 技术参数 → 用户场景语言
- 例：`4000+lm` → "卤素灯能照 X 米，我们的灯能照 Y 米"
- 删除晦涩难懂的名词
- 只保留：板块、文案、目的、设计需求、整体调性

### 官网裁剪参考
- 基于 sealight-led.com 判断冗余板块
- 三合一页不重复官网已有的通用信息

## 需求文档输出要求

### 格式
- 精简版（小白可读）
- 供团队审核后交技术部门
- 明确：板块 / 文案 / 目的 / 设计需求（图片）/ 整体调性需求

### 后续对接
- 场景划分 + 痛点（中文）→ 开会讲"为什么是这个场景"
- 初步判断可选系列 → 开会讨论
- SEO FAQ 等新增板块 → 他人负责
